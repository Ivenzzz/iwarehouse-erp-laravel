# Plan: Update Product Masters CSV Export Columns

## Goal
Modify the CSV export in `ProductMastersPage` to include specific columns: Brand, Model, Category, Subcategory, RAM, ROM, Color, Model Code, CPU, GPU, RAM Type, ROM Type, OS, Screen.

## Current State
- Export is at **ProductMaster** level (one row per master)
- Exports columns: master_sku, brand, model, category, subcategory, image_url, description, + all spec definitions
- Spec definitions contain: platform_cpu, platform_gpu, platform_os
- **Missing**: RAM, ROM, Color, Model Code, RAM Type, ROM Type, Screen

## Issue
The requested columns (RAM, ROM, Color, Model Code, RAM Type, ROM Type, Screen) are stored in **ProductVariant** model, not ProductMaster. The export currently iterates ProductMaster records but needs to iterate ProductVariant records instead.

## Implementation

### 1. Modify ExportProductMastersCsv.php

**File:** `app/Features/ProductMasters/Actions/ExportProductMastersCsv.php`

**Changes:**
1. Change query from ProductMaster to ProductVariant
2. Filter: `->where('is_active', true)`
3. Load necessary relations: productMaster.model.brand, productMaster.subcategory.parent
4. Replace fixed columns with user-requested columns:
   - Brand → `$variant->productMaster->model->brand->name`
   - Model → `$variant->productMaster->model->model_name`
   - Category → `$variant->productMaster->subcategory->parent?->name`
   - Subcategory → `$variant->productMaster->subcategory->name`
   - RAM → `$variant->ram`
   - ROM → `$variant->rom`
   - Color → `$variant->color`
   - Model Code → `$variant->model_code`
   - CPU → `$variant->cpu`
   - GPU → `$variant->gpu`
   - RAM Type → `$variant->ram_type`
   - ROM Type → `$variant->rom_type`
   - OS → `$variant->operating_system`
   - Screen → `$variant->screen`
4. Replace `$specKeys` spec definitions with direct variant attributes
